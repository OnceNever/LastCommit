# LastCommit

> Not the last line of code, but the last explanation.  
> 不是最后一行代码，而是最后一次解释。

**LastCommit** 是一个用于生成「代码遗嘱（Code Will）」的工具。  
它帮助即将交接、离开或暂时放下项目的开发者，用**人话**告诉后来者：

- 这个项目真正是怎么运作的
- 哪些地方千万别乱改
- 现在看到的混乱是怎么一步步变成这样的
- 如果一定要重构，第一刀该从哪下

---

## ✨ 为什么需要 LastCommit？

大多数项目交接文档都失败在同一件事上：

> **它们解释了“是什么”，却没解释“为什么”。**

而真正致命的，恰恰是这些没人写下来的信息：
- 当初为什么不用更优雅的方案？
- 哪段代码是历史妥协，哪段是核心设计？
- 哪些坑你明知道，但没时间填？

LastCommit 试图把这些**“只存在于作者脑子里的上下文”**留下来。

---

## 🧠 LastCommit 会做什么

LastCommit 会分析你的项目，并生成一份结构化的「代码遗嘱」，内容包括：

- 📌 **项目人话版简介**  
  用非 README 的方式解释项目的真实目标

- 🧱 **核心设计说明（不要轻易动）**  
  哪些模块是地基，改了会发生什么

- 🧨 **已知问题 & 技术债**  
  不粉饰太平，只说真实情况

- 🕰️ **历史背景 & 需求演变**  
  为什么代码会变成现在这样

- 🗺️ **如果你一定要重构**  
  推荐顺序、安全切入口、禁区提醒

- 💬 **留给接手者的一句话**
